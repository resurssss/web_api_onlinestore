# Лабораторная работа: Настройка Nginx, TLS и балансировки нагрузки

## Цели работы
- Настроить Nginx для обратного проксирования запросов к нескольким экземплярам сервера
- Реализовать TLS для шифрования соединений
- Организовать раздачу статических файлов через Nginx
- Настроить round-robin балансировку нагрузки между серверными инстансами
- Проверить идентификацию обработавшего запрос инстанса

## Выполненные задачи

### 1. Три Docker-инстанса сервера
- Созданы три экземпляра приложения: app1, app2, app3
- Каждый инстанс имеет уникальный идентификатор (Instance-1, Instance-2, Instance-3)
- Идентификатор передаётся в HTTP-заголовке `X-Instance-Id`

### 2. Конфигурация Nginx
- Создан upstream блок с тремя серверами
- Используется round-robin стратегия балансировки
- Настроен proxy_pass для проксирования запросов
- Прямой доступ к серверам закрыт (порты не проброшены)

### 3. Раздача статических файлов
- Настроен location /static/ для отдачи файлов без проксирования
- Тестовый файл static/style.css успешно отдаётся

### 4. TLS с самоподписанным сертификатом
- Сгенерирован сертификат и ключ через OpenSSL
- Настроен HTTPS на порту 443
- HTTP (порт 80) перенаправляет на HTTPS
